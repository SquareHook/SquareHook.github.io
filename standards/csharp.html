C# Coding Standards

As a team, we have agreed upon conventions and code standards for C#. Following these standards will help to make your code more consistent readable for other developers. Please follow and enforce the standards below while writing code and doing peer reviews.

Comments

Comments should be placed throughout your code, especially in areas that are not easily understood at a glance. At the bare minimum, you should write a comment above each method. Use triple forward slashes to generate a comment that will show in Visual Studio auto-complete.

Variable Names 

    
        //namespaces are upper camel case
        namespace DataModels
        {
            //class names should be upper camel case
            public class Widget
            {
                //private member variables should be lower camel case
                private string name;
                private int productNumber;
        
                //public member variables are upper camel case
                public string CatalogNumber { get; }
        
                //method parameters are upper camel case
                public Widget(string name, int productNumber)
                {
                    this.name = name;
                    this.productNumber = productNumber;
                    this.CatalogNumber = name + productNumber;
                }
        
                //public methods are upper camel case
                public void UpdateProductNumber(int productNumber)
                {
                    this.productNumber = productNumber;
                }
        
                //private methods are lower camel case
                private void doSomething()
                {
                }
            }
        }
        
        //constants should be all uppercase
        const double TAXRATE = 6.650;
